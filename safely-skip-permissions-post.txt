We built --safely-skip-permissions. It's called Cortex Code.

Every coding agent has some version of --dangerously-skip-permissions. You flip it, the agent runs unattended, and one bad DROP DATABASE prod later you're writing a postmortem.

The whole industry is trying to solve this at the agent layer. Prompt guards, confirmation dialogs, permission systems bolted onto the tool. That's the wrong layer. The agent isn't the control plane. The execution environment is.

Cortex Code has /bypass -- auto-approves every tool call, no confirmation prompts. I use it constantly. I don't worry about it because the agent runs inside Snowflake. The platform already enforces the boundaries.

RBAC -- scope the agent to a role with SELECT on dev and USAGE on a warehouse. It physically cannot drop production. The SQL returns an insufficient privileges error.

UNDROP + Time Travel -- agent truncates a table in your sandbox? UNDROP TABLE. Query the data as it existed 30 minutes ago with AT(OFFSET => -1800). Up to 90 days of Time Travel retention if you configure it.

ACCESS_HISTORY / QUERY_HISTORY -- every statement the agent runs is logged, attributable, and auditable.

Well-scoped role + reversible environment + full audit trail = let the agent work.

The governance model that protects you from a junior engineer running the wrong query already protects you from an agent doing it. Bolting safety onto the agent layer will never fully solve the problem. The execution environment has to enforce it.

--safely-skip-permissions is just a well-scoped environment.
